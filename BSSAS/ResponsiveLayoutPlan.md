# BSSAS QML 响应式布局实施方案

## 目标

本次适配以 2520x1680 开发屏幕为设计基准，保持现有 `screenwidthratio: 0.765` 和 `screenaspectratio: 16 / 10` 不变。当前基准窗口约为 1928x1205；1920x1080 屏幕下默认窗口约为 1469x918，是本轮最低正式验收窗口。

目标不是对整棵 QML 做 `Scale` 缩放，而是使用 Qt Quick Layouts、断点、最小/最大约束、垂直滚动和局部重排，保证小一档窗口中不重叠、不裁切关键内容、不依赖水平滚动兜底。

## 设计原则

- 保留大屏基准：1928x1205 及以上继续走 expanded 分支，避免改变开发屏上的视觉比例。
- 使用统一断点：全局通过 `Constants.isCompactContent(width, height)` 和 `Constants.isNarrowContent(width)` 判断内容区域，而不是各页面各自猜测。
- 优先重排，其次滚动：信息密集页在 compact 下改为更少列数或上下布局；内容仍然超高时只允许垂直滚动。
- 控件不无限缩小：输入框、按钮、复选框、开关、图标按钮保持可读和可点尺寸，由父布局控制可用宽度。
- 图表不整体缩放：波形、频谱和定位图继续由容器尺寸驱动，保留 `anchors.fill` 与内部绘制逻辑。

## 实施范围

- `BSSAS/Constants.qml`：增加基准窗口、最低窗口、compact/narrow 断点和 helper。
- `BSSASContent/App.qml`：增加最低窗口尺寸和根级 compact 状态。
- 主壳层：背景内容区、左侧导航、设置侧栏根据 compact 状态收紧边距、圆角和导航尺寸。
- 页面层：首页详情区、软件总控页、DAQ/信号处理/文件管理子页、传感器监测、时频监测、病历管理、辅助诊断、关于页接入统一断点。
- 弹窗层：日期/时间选择器按可用 overlay 尺寸限制宽高，避免 1920x1080 下越界。

## 页面策略

- 首页：hero 和概览卡保留现有响应式逻辑；下方详情区在 compact 下从横向双栏改为单列栅格并允许垂直滚动。
- 软件总控：标题、TabBar、内容区边距在 compact 下收紧，Tab 文本使用 elide 防止挤压。
- DAQ、信号处理、文件管理：保留 ScrollView，收紧页面边距、卡片边距和控制项宽度；已有 GridLayout 继续按可用宽度降列。
- 传感器监测：增加外层 ScrollView；compact/narrow 下腹部定位图与点位列表上下排列，点位列表内部继续滚动。
- 时频监测：compact 下左侧控制栏宽度、卡片高度、按钮高度和 Tab 字号收紧，图表区继续占满剩余空间。
- 病历管理：保留表单 ScrollView；标题在 compact 下允许两行并 elide；表单 GridLayout 继续按宽度降为单列。
- 辅助诊断：compact 阈值提升到统一断点；顶部操作区单列，结果区和摘要区在 compact 下上下排列，并由页面 ScrollView 承载。
- 关于页：compact 阈值提升到统一断点；信息卡和设计价值区在 1920x1080 默认窗口下使用更稳的两列布局。

## 验收清单

- 构建：`cmake --preset debug` 和 `cmake --build --preset debug` 通过。
- 2520x1680：默认窗口下检查所有主页面和软件总控四个子页，确认原有视觉比例基本不变。
- 1920x1080：默认窗口下检查所有主页面和关键弹窗，确认无重叠、无关键内容裁切、无水平滚动兜底。
- 全屏/还原：两个目标分辨率下切换全屏与还原，内容区、设置侧栏、Toast、Popup 位置正常。
- 交互：导航切页、Tab 切换、表单输入、传感器坐标滚轮微调、日期时间选择、文件导入保存、辅助诊断模型菜单可正常操作。
